Student Portal — Android + Django REST + MongoDB

Version 1.0

1. Overview

Comprehensive technical and product documentation for the Student Portal project. The system comprises an Android application (Kotlin + Jetpack Compose) and a Django REST Framework backend with MongoDB for domain data and SQLite for Django internals. Modules include authentication, profile, academics, finance, library, virtual campus, and a voting system.

2. Architecture

  • Android App: Kotlin, Jetpack Compose, Navigation-Compose, Room, DataStore, Retrofit/OkHttp, kotlinx.serialization, Timber
  • Backend: Django 5, DRF, JWT (SimpleJWT), CORS, SQLite (Django), MongoDB (domain data)
  • Data: MongoDB collections for users, profiles, academics, finance, voting; Room for offline cache
  • Video: Google Meet (Intent-based) integrated; optional Zoom SDK via gradle properties

Flow: Compose UI → Repositories → Retrofit ApiService → Django REST → MongoDB; tokens in DataStore via OkHttp interceptor.

3. Repository Structure

android/              # Android app root
  app/                # App module (Compose UI, navigation, data layer)
backend/              # Django REST backend (DRF + MongoDB)

4. Environments & Dependencies

  • Android: AGP 8.9.1, Kotlin 2.0.21, Compose BOM 2024.09.00, compileSdk 35, minSdk 24
  • Libraries: Room 2.6.1, DataStore 1.1.1, Retrofit 2.11.0, OkHttp 4.12.0, kotlinx-serialization 1.7.3
  • Backend: Django 5.1.1, DRF 3.15.2, django-cors-headers 4.4.0, SimpleJWT 5.3.1, PyMongo 4.6.1, argon2-cffi 23.1.0

5. Setup & Local Development

Prereqs: Android Studio (Giraffe+), JDK 11, Python 3.10+, MongoDB at localhost:27017.

Backend

python -m venv backend\.venv
backend\.venv\Scripts\Activate.ps1
pip install -r backend\requirements.txt
cd backend
python manage.py migrate
python manage.py create_sample_data     # optional
python manage.py create_voting_indexes  # once
python manage.py runserver 0.0.0.0:8000

Android

  • Base URLs in NetworkModule.kt: http://10.0.2.2:8000/api/, http://<LAN-IP>:8000/api/, http://127.0.0.1:8000/api/
  • Open in Android Studio and run on emulator/device

6. Configuration

Android: INTERNET, ACCESS_NETWORK_STATE, CAMERA, RECORD_AUDIO; cleartext allowed for dev via networkSecurityConfig.

Backend: DEBUG=True, permissive CORS for local dev, JWT auth, SQLite (Django) + MongoDB (domain), structured console logging.

7. Android Application

  • UI: Compose screens for Login, Home, Profile, Academics, Finance, Library, Virtual Campus, Voting
  • Data: ApiService, NetworkModule (OkHttp + JSON), repositories (AuthRepository, ProfileRepository, VotingRepository)
  • Storage: Room (Profile cache), DataStore (auth token)
  • Networking: Fallback base URLs; Authorization header added when token present
  • Video: Google Meet integrated; optional Zoom via gradle properties

8. Backend API

Base: /api/

  • Auth: POST /api/auth/jwt/login, POST /api/auth/jwt/refresh
  • Custom: POST /api/auth/login, POST /api/auth/logout, GET /api/profile/me
  • Academics: GET /api/academics/course-registration, /course-work, /exam-card, /exam-audit, /exam-result, /academic-leave, /clearance, /insights
  • Finance: GET /api/finance/fee-payment, /view-balance, /receipts
  • Virtual: GET /api/virtual/dashboard, /my-courses, /lectures, /zoom-rooms; POST/GET attendance endpoints
  • Voting: GET/POST /api/voting/elections, GET /api/voting/elections/{id}, POST candidates/vote, GET results

9. Data Model (MongoDB)

  • voting_elections: { _id, title, candidates: [ { name } ], results: { <candidate>: votes }, created_at, updated_at }
  • voting_votes: { election_id, user_id, candidate, created_at } — unique index (election_id, user_id)
  • attendance_records: { room_id, course_id, user_id, date, marked_at }

10. Security & Authentication

  • JWT via SimpleJWT; Authorization: Bearer token
  • Password hashing with Argon2 preferred
  • CORS enabled for local development and emulator hosts

11. Observability & Logging

  • Backend: structured console logs with environment-level
  • Android: OkHttp logs via Timber (DEBUG only)

12. Testing

  • Android: unit and instrumentation tests
  • Backend: Django management commands and sample tests

13. Deployment

  • Backend: containerize, configure env, create indexes, HTTPS via Nginx
  • Android: environment-aware base URLs, signed release builds

14. Roadmap

  • Wire remaining Android screens to backend APIs
  • Offline-first sync and robust error states
  • RBAC, audit logging, rate limiting

  • Zoom SDK feature flag with fallback
  • CI/CD: lint, test, build, release

15. Troubleshooting

  • Emulator → backend: use http://10.0.2.2:8000/api/
  • 401 login: confirm MongoDB running and sample data created
  • 409 vote: unique constraint prevents duplicate votes
  • CORS: ensure origin/device host are permitted
