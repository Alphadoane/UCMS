ABSTRACT
The Integrated Campus Management System (ICMS) is a comprehensive digital solution designed to streamline key administrative and academic functions at KCA University. The system will integrate four major modules—Student Portal, Virtual Campus, Complaint Management, and Fee Billing—into a unified platform accessible through mobile interfaces. The ICMS aims to enhance institutional efficiency, transparency, and communication by automating routine tasks and facilitating real-time access to academic and financial data.
Through the Student Portal module, learners can register, access course materials, and view academic progress. The Virtual Campus feature fosters online learning and engagement between students and lecturers, while the Complaint Management System will provide an efficient channel for students to lodge and track grievances. The Fee Billing module automates invoicing, payments, and receipt generation, ensuring accurate and timely financial management.
The project adopts an object-oriented design methodology using the waterfall development model, emphasizing structured phases of requirement gathering, system design, implementation, testing, and deployment. The backend will be powered by a relational database, MySQL, while the frontend utilizes Android Studio and visual studio code for mobile development.
Overall, the ICMS will bridge existing gaps in communication, data management, and service delivery within the university. It is envisioned to promote digital transformation, reduce paperwork, and improve user satisfaction by offering a centralized, secure, and scalable University management environment.

SECURITY AND PRIVACY

Security Measures
User Authentication and Authorization

The system uses JWT (JSON Web Token) authentication for secure login and session management.
Users must provide valid credentials (email/username and password) before accessing restricted services.
Passwords are never stored in plain text; they are hashed using PBKDF2 or Argon2 via Django’s authentication system.
Role-based access control (RBAC) ensures that users only access resources permitted for their role (e.g., Admin, Customer, or Manager).

Secure Communication

All communication between the Android client and Django server is transmitted over HTTPS using SSL/TLS encryption.
API endpoints enforce authentication headers to prevent interception or replay attacks.
Sensitive data such as tokens or API keys are stored securely using Android Keystore and never hardcoded in the source code.

Database Security (MySQL)

The MySQL database is secured with strong credentials and restricted access privileges.
Connections between the Django server and MySQL are established using encrypted SSL connections.
Prepared statements and ORM queries are used to prevent SQL Injection attacks.
Regular database backups and data recovery mechanisms are implemented to ensure data availability.
Data at rest is protected through encryption and hashed identifiers for sensitive information.

API Security

All API endpoints are protected using token-based authentication and rate limiting to prevent brute-force or denial-of-service attacks.
Input validation and server-side sanitization prevent malicious payloads (e.g., cross-site scripting (XSS) or JSON injection).
APIs are version-controlled and use CORS (Cross-Origin Resource Sharing) settings to limit access only to authorized client origins.
Error messages are sanitized to avoid disclosing system internals.

Data Validation and Sanitization

All user input is validated both on the Android client and the Django backend to prevent malformed data or code injection.
Special characters, file uploads, and form data are sanitized before database storage.
Validation ensures data integrity and prevents cross-site scripting (XSS) and buffer overflow vulnerabilities.

Logging and Monitoring

All major system activities, logins, and data transactions are logged for audit purposes.
Logs are secured to prevent tampering and are stored with timestamps and user identifiers.
Automated monitoring tools can detect abnormal behavior such as multiple failed login attempts or suspicious API requests.

Error Handling and Exception Management

The system implements safe error handling to ensure that error messages do not reveal technical details about the server or database.
All exceptions are caught and logged securely without exposing stack traces to end-users.
User-friendly messages are displayed on the Android application for better experience and security.

Regular Security Updates

Django and Android dependencies are kept up-to-date to address the latest security vulnerabilities.
Regular security audits and penetration tests are scheduled to identify and fix weaknesses.
The development team adheres to OWASP Mobile and Web Security Guidelines.

Privacy Protection

Data Minimization

Only essential user information (e.g., name, email, delivery address) is collected.
No unnecessary personal data or sensitive identifiers are requested from users.

User Consent

The app includes a Privacy Policy that clearly explains data collection, usage, and sharing practices.
Users must explicitly accept permissions before accessing location, storage, or camera services.

Data Storage and Retention

User data is stored securely on the Django server and MySQL database, not on the Android device.
Personal data is retained only as long as necessary for operational or legal reasons.
Users can request account deletion or data removal in compliance with data protection principles.





Data Sharing

User data is never sold or shared with third parties without consent.
Any integration with third-party APIs (e.g., payment gateways or Google Maps) follows secure OAuth authentication mechanisms.

Compliance

The system complies with global privacy standards such as GDPR (General Data Protection Regulation) and Kenya Data Protection Act 2019.
Privacy settings are transparent and user-controlled.

TESTING STRATEGY

The types of testing strategies implemented are:
Functional Testing
Ensures that each function of the system operates according to requirements.
Performance Testing
Measures the responsiveness and stability of the application under load.
Tools: Apache JMeter, Locust, or Django’s testserver stress testing.
Objectives:
Verify API response times (< 2 seconds per request).
Evaluate server scalability under concurrent user load.
Security Testing
Ensures the system is protected against unauthorized access and data breaches.
Tests include:
Authentication and authorization checks (JWT token validation).
SQL Injection and XSS attack prevention.
Encrypted communication (HTTPS).
Tools: OWASP ZAP, Burp Suite.

Compatibility Testing
Ensures the app runs consistently across different devices, OS versions, and screen sizes.
Tools: Android Studio Emulator, Firebase Test Lab.
Test Scenarios:
Android 10 → Android 15 devices
Phones and tablets with varied resolutions

Regression Testing
Conducted after each update or bug fix to ensure existing functionality is not broken.
Tools: Jenkins for CI/CD test automation.

Database Testing
Validates MySQL data integrity and backend consistency.
Focus:
Schema validation
Data relationships and foreign key constraints
Query performance and transaction rollback
